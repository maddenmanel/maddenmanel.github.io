## 一、一个扎心的事实

打开你的微信收藏夹。上一次真的回去翻看一篇"AI 教程"，是什么时候？

我猜是没有。

不是你懒。是整个 AI 世界太吵了——每天都有"震撼发布"，每周都有"颠覆性模型"，每个号都在喊"不学就被淘汰"。结果呢，你存了 500 篇文章，装了 20 个插件，试了 10 个工具，真正留在你工作流里的不超过 3 个。

今天这篇不是又一个"必读清单"。今天只告诉你一件事：**停止收藏，开始让 AI 替你整理。**

## 二、Karpathy 的方法

这个方法来自前 OpenAI 联合创始人 Andrej Karpathy。他几天前公开了自己的个人知识库系统，用这套方法在一个研究主题上积累了约 100 篇笔记、40 万字，随时可以查询、可以提问。

核心思路只有一句话：

> 你只负责"丢进去"，AI 负责"整理好"。

传统笔记工具（Notion、印象笔记、Obsidian）的逻辑是甩给你一个空壳，剩下的都是人力活——贴标签、建链接、找矛盾、维护目录。这套"维护成本"就是大多数人放弃的真正原因。

Karpathy 把它反过来：**原始素材只进不改，整理由 AI 增量完成，人负责提问和判断。**

## 三、需要的三样东西

1. **Obsidian**。一个把笔记存成本地 Markdown 文件的软件。所有数据在你自己电脑上。免费，到 https://obsidian.md 下载。

2. **Obsidian Web Clipper**。浏览器插件，一键把网页文章存进 Obsidian。免费，在浏览器应用商店搜名字即可。

3. **Claude Code**。这是整套系统的关键。

前两个没什么可说的，重点说第三个。

ChatGPT 或 Claude 网页版只能在对话框里聊天，你得手动复制粘贴。Claude Code 不一样，它可以直接打开你的文件夹，读你的笔记、写新笔记、改目录——像一个真的坐在你电脑前帮你干活的助理。

Claude Code 约 20 美元/月，是这套方案里唯一要花钱的部分。如果你一个月用 AI 的时间超过 5 小时，这笔钱比喝星巴克还划算。

## 四、关键的一段指令

打开 Claude Code，让它指向你刚建好的 Obsidian 文件夹，整段粘贴下面这段话：

```
请在当前文件夹帮我搭建一个个人 AI 知识库系统，
架构参考 Karpathy 的方案：

三个核心区域：
raw/  我丢进来的所有原始素材（网页、PDF、截图），只进不改
wiki/ 你帮我整理的笔记，每个概念一篇，互相链接
index.md + log.md  目录和整理记录，由你自动维护

四个工作阶段：
收集：我用 Obsidian Web Clipper 存到 raw/
编译：你读取 raw/ 的新资料，为每个概念生成或更新笔记
查询：我对知识库提问，你的回答也要存回 wiki/
维护：定期扫描整个知识库，找矛盾、补缺漏

要求：所有笔记使用简体中文，建好文件夹结构和初始模板，
完成后告诉我第一步该做什么。
```

就这样。Claude Code 会自动把骨架全部搭好，你一行代码都不用写。

## 五、每天做什么

建好之后，日常只有四个动作。

**收集**。看到好东西就往 `raw/` 里丢。浏览器插件点一下，或者把 PDF 直接拖进去。这个文件夹只进不改，是原始档案。

**整理**。每周一次。对 Claude Code 说一句"请读取 `raw/` 里的新资料，更新 `wiki/` 和目录"。泡杯咖啡回来，笔记就都整理好了。

**提问**。有问题随时问。比如"帮我整理 XX 主题的核心观点"、"找出这些笔记里互相矛盾的地方"、"基于这些资料写份给老板的汇报提纲"。关键技巧：**让它把答案也存回 `wiki/`**。每一次提问都在给知识库添砖加瓦，而不是问完就没了。

**清理**。每月一次。"请扫描整个知识库，找出矛盾、过时信息，并建议下一步可以研究什么。" AI 会反过来推着你学——这才是知识复利。

## 六、两个冷静的提醒

别被"100 篇笔记 40 万字"吓到。Karpathy 自己强调过两件事，我觉得比方法本身更重要。

**第一，这套方法只适合"个人规模"。** 如果你的资料超过 1 万篇，这套就不够用了，得上向量数据库。但 99% 的人永远到不了那个量级，别被"企业级 RAG 系统"的营销话术吓到。

**第二，AI 整理不等于你就不用看了。** AI 能给你结构化的笔记，但判断力还是你自己的。它能省掉 80% 的整理时间，但省不掉 20% 的思考时间。如果你把它当成思考的替代品，这套系统会变成新的收藏夹，一样吃灰。
